# S03 – eda_cli: мини-EDA для CSV

Небольшое CLI-приложение для базового анализа CSV-файлов.
Используется в рамках Семинара 03 курса «Инженерия ИИ».

## Требования

- Python 3.11+
- [uv](https://docs.astral.sh/uv/) установлен в систему

## Инициализация проекта

В корне проекта (S03):

```bash
uv sync
```

Эта команда:

- создаст виртуальное окружение `.venv`;
- установит зависимости из `pyproject.toml`;
- установит сам проект `eda-cli` в окружение.

## Запуск CLI

### Краткий обзор

```bash
uv run eda-cli overview data/example.csv
```

Параметры:

- `--sep` – разделитель (по умолчанию `,`);
- `--encoding` – кодировка (по умолчанию `utf-8`).

### Полный EDA-отчёт

```bash
uv run eda-cli report data/example.csv --out-dir reports


```md
#### Дополнительные параметры команды report

Команда `report` поддерживает дополнительные параметры, влияющие на содержимое отчёта:

- `--max-hist-columns`  
  Максимальное количество числовых колонок, для которых строятся гистограммы.
  Если числовых признаков больше, гистограммы строятся только для первых N.

- `--top-k-categories`  
  Количество наиболее частых значений (top-k), сохраняемых для каждого
  категориального (строкового) признака.

- `--title`  
  Заголовок отчёта. Используется как основной заголовок (`# ...`)
  в файле `report.md`.

Пример запуска с дополнительными параметрами:

```bash
uv run eda-cli report data/example.csv \
  --out-dir reports \
  --max-hist-columns 4 \
  --top-k-categories 10 \
  --title "EDA отчёт по данным"
```

В результате в каталоге `reports/` появятся:

- `report.md` – основной отчёт в Markdown;
- `summary.csv` – таблица по колонкам;
- `missing.csv` – пропуски по колонкам;
- `correlation.csv` – корреляционная матрица (если есть числовые признаки);
- `top_categories/*.csv` – top-k категорий по строковым признакам;
- `hist_*.png` – гистограммы числовых колонок;
- `missing_matrix.png` – визуализация пропусков;
- `correlation_heatmap.png` – тепловая карта корреляций.

Проверки качества данных

В рамках формирования отчёта дополнительно реализованы проверки качества данных, позволяющие выявлять потенциальные проблемы в структуре датасета. Результаты проверок представлены в виде логических флагов.

Проверка наличия константных колонок

Функция has_constant_columns определяет, присутствуют ли в датасете колонки, в которых все значения одинаковые.

Данная проверка позволяет обнаружить признаки, не несущие аналитической ценности, так как константные колонки не влияют на результаты анализа и моделирования.

Проверка дубликатов идентификаторов

Функция has_suspicious_id_duplicates предназначена для проверки уникальности идентификаторов (например, id, user_id).

Наличие дубликатов в идентификаторах может указывать на ошибки при формировании данных, повторяющиеся записи или некорректное объединение таблиц.
